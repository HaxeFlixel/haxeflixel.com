I found something more.

It can be used for a temporary solution to fix the bug, however I need to modify 3 classes.

 

1. FlxMouseButton.hx

This line, which I think it causes the bug since it evaluates between pressed mouse button (from current == JUST_PRESSED) or (from current == JUST_RELEASED) 

inline public function justPressed():Bool { return (current == JUST_PRESSED || current == JUST_RELEASED); }

So, I added a new method which only evaluate the current == JUST_PRESSED only. (Yeah, I didn't delete the original method to prevent any weird effect)

inline public function justPressed():Bool { return (current == JUST_PRESSED || current == JUST_RELEASED); } inline public function justPressedOnly():Bool { return (current == JUST_PRESSED); }

 

2. FlxMouse.hx

I added more method to FlxMouse class since FlxG uses FlxMouse instead of FlxMouseButton

public function justPressedOnly():Bool { return _leftButton.justPressedOnly(); }

 

3. MouseEventManager.hx

I replaced FlxG.mouse.justPressed in from this line

if (FlxG.mouse.justPressed) { for (current in currentOverSprites) { if (current.onMouseDown != null) { current.onMouseDown(current.sprite); } } }

 

into 

if (FlxG.mouse.justPressedOnly()) { for (current in currentOverSprites) { if (current.onMouseDown != null) { current.onMouseDown(current.sprite); } } }

 

It solves the bug, which the event doesn't get triggered again when I released the mouse after I clicked the sprite. 

Hope this help temporarily, but I haven't tested for any side effect yet.

 

EDIT:

However, this is only for temporarily fixing the double click bug. 

There was one another, which the event is triggered even when the sprite is not visible or added into FlxGroup or FlxState.

 

EDIT:

Temporary solution I use to make the event not triggered when the sprite is not visible is not check whether the sprite's variable "exists" is true or not (not sure whether should I choose "visible" or "exists". If I choose both, it will increase the number of operation by 2 instead of one [Ok, that's is just an extra two operation. Not very much xD...] ).

 

So, in MouseEventManager I added "&& current.sprite.exists" from update() method at line

(current.onMouseDown != null)

which becomes

(current.onMouseDown != null && current.sprite.exists)

It works assuming that the sprite is already added into the group/state (it won't work if the sprite is not added yet into the group/state).

Gama11
4 months 1 week ago

@Vsio: I think checking for visibility and existance here makes a lot of sense. I added these checks for all the MouseEventManager callbacks. :)

@JimmyDeemo: "alpha" and "visible" are actually completely independent. visible controls whether a sprite is rendered - setting visible = false; on a sprite thus reduces the amount of draw calls (you can see those in the perf window of the debugger). A sprite with alpha = 0, however, will still be rendered!  This means you should never just set alpha to 0 when all you actually just want to hide the sprite - use visible for that!

Analogously to visible controlling whether draw() gets called, "active" controls whether update() is called.

Okay, I have finally made a tweak and it works well except I haven't tested its safety yet (against exception and side effect). Thanks for Gama11's "parent" clue. It gave me idea xD. But yeah, the weakness is at the "same sprite to multiple groups").

(Personally, I usually don't use same sprite to in multiply grup unless removing it first xD. Feel free to modify it again if the better one has been found xD)

 

So, the mouse event will be triggered only when the sprite is really visible which from all its parent and itself, it needs all to be true (its "visible" and "exists" variable) to be triggered. So, here's what I do:

 

1. In FlxBasic.hx, add parent field

public var _parent:FlxBasic;

 

2. In FlxTypedGroup.hx, I modified its add and remove function

public function add(Object:T):T { if (Object == null) { FlxG.log.warn("Cannot add a `null` object to a FlxGroup."); return null; } // Don't bother adding an object twice. if (FlxArrayUtil.indexOf(members, Object) >= 0) { return Object; } // First, look for a null entry where we can add the object. var i:Int = 0; var l:Int = members.length; while (i < l) { if (members[i] == null) { members[i] = Object; if (i >= length) { length = i + 1; } Object._parent = this; return Object; } i++; } // Failing that, expand the array (if we can) and add the object. if (maxSize > 0) { if (members.length >= maxSize) { Object._parent = this; return Object; } else if (members.length * 2 <= maxSize) { FlxArrayUtil.setLength(members, members.length * 2); } else { FlxArrayUtil.setLength(members, maxSize); } } else { FlxArrayUtil.setLength(members, members.length * 2); } // If we made it this far, then we successfully grew the group, // and we can go ahead and add the object at the first open slot. members[i] = Object; length = i + 1; Object._parent = this; return Object; } public function remove(Object:T, Splice:Bool = false):T { if (members == null) { return null; } var index:Int = FlxArrayUtil.indexOf(members, Object); if ((index < 0) || (index >= members.length)) { return null; } if (Splice) { members.splice(index, 1); } else { members[index] = null; } Object._parent = null; return Object; }

 

3. In MouseEventManager.hx, I added checkSpriteVisibility function and modified its update() function:

- checkSpriteVisibility (sprite_:FlxBasic)

public function checkSpriteVisibility(sprite_:FlxBasic):Bool { var tree:FlxBasic = sprite_; while (tree != null) { if (tree.visible && tree.exists) { // Do nothing } else { return false; } tree = tree._parent; } return true; }

 

update()

override public function update():Void { super.update(); var currentOverSprites:Array<SpriteReg> = new Array<SpriteReg>(); for (reg in _registeredSprites) { // Sprite destroyed check. if (reg.sprite.scale == null) { removeSprite(reg.sprite); continue; } if (reg.sprite.alive == false || reg.mouseEnabled == false ) { continue; } if (checkOverlap(reg.sprite)) { currentOverSprites.push(reg); if (reg.mouseChildren == false) { break; } } } // MouseOver - Look for new sprites with mouse over. for (current in currentOverSprites) { if (getRegister(current.sprite, _mouseOverSprites) == null) { if (current.onMouseOver != null && checkSpriteVisibility(current.sprite)) { current.onMouseOver(current.sprite); } } } // MouseOut - Look for sprites that lost mouse over. for (over in _mouseOverSprites) { if (getRegister(over.sprite, currentOverSprites) == null) { if (over.onMouseOut != null && checkSpriteVisibility(over.sprite)) { over.onMouseOut(over.sprite); } } } // MouseDown - Look for sprites with mouse over when user presses mouse button. #if !FLX_NO_MOUSE if (FlxG.mouse.justPressed) { for (current in currentOverSprites) { if (current.onMouseDown != null && checkSpriteVisibility(current.sprite)) { current.onMouseDown(current.sprite); } } } // MouseUp - Look for sprites with mouse over when user releases mouse button. if (FlxG.mouse.justReleased) { for (current in currentOverSprites) { if (current.onMouseUp != null && checkSpriteVisibility(current.sprite)) { current.onMouseUp(current.sprite); } } } #end _mouseOverSprites = currentOverSprites; }

 

Well, that's my tweaking. Once again, it works well, but I haven't tested its safely yet especially the "_parent" value will only change at add or remove function.  Not sure yet with other function.
